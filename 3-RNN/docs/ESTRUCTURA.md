# Estructura del proyecto

```text
3-RNN/
├── dataset/        # Corpus de codigo en C
├── docs/           # Documentacion
├── models/         # Pesos y artefactos entrenados
├── notebooks/      # Experimentos y entrenamiento en notebook
├── src/            # Scripts de preprocess, training, api
└── README.md
```

## Convenciones

- Mantener nombres y estilo de codigo consistentes en el dataset.
- Guardar modelos con versionado simple (ej. rnn_v1.h5).
- Documentar cambios importantes en `docs/`.
